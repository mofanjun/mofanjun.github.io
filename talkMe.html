# 20190823001：spss 无法导入 office 2016 access 文件

# 问题描述:

spss 导入数据库文件 -> 添加ODBC数据源 -> 编辑 时报 找不到Microsoft Access Driver(*.mdb)ODBC驱动程序的安装例程。按照csdn和cnblog上的文章并无法解决我的问题。

# 解决方案:

需要额外安装ODBC连接文件，其次你得看你的office是32位或64位，然后去下载对应驱动，附上微软原文: 无法在 Office 即点即用应用程序之外使用 Access ODBC 驱动程序或 OLEDB 提供程序

# 20190823002：spss 从Excel复制中文数据乱码。

# 问题描述：

当我从Excel中复制数据，使用spss25中的与变量名一起粘贴功能后，数据显示□，无法正常显示中文数据。

# 解决方法：

这是因为spss对本质上还是spss对双字节文本的支持有bug，非乱码而是数据截断，可以分两步走：

  1. 将变量名和一行数据一同粘贴，然后使用与变量名一起粘贴功能，在变量视图调整宽度，然后删除截断数据。
  2. 将剩余数据粘贴到数据视图中。

吐槽

这是个取巧的办法，不是非得和我一样操作，但是因为spss对中文不是很支持，预先对变量设置宽度肯定是需要的。那在操作中文字符时与变量名一起粘贴功能，就显得可有可无了。
